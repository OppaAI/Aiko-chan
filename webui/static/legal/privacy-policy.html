Privacy Policy — Aiko-chan (OppaAI)

Last updated: Jul 07, 2026

Aiko-chan is an independent, self-hosted AI companion project run by a single developer (OppaAI). This policy explains what data is collected when you use it and how it's handled.

1. Who runs this

Aiko-chan is operated by an individual developer, not a company. Infrastructure runs on personally-owned hardware (Jetson + PC). There is no dedicated support team, and this policy may be updated informally as the project evolves.

2. What data is collected

Account / identity data - If you sign in with Patreon, we receive your Patreon member ID and membership/pledge status (used only to verify paid access). - If you sign in with GitHub, we receive your GitHub username and public profile info (used only to verify contributor access). - We do not receive your Patreon or GitHub password — authentication happens via OAuth, and credentials never pass through our servers.

Conversation data - Messages you send to Aiko, and Aiko's responses, are logged. - Conversations are used to build short-term and long-term memory (e.g. semantic embeddings stored in a local vector database) so Aiko can recall context across sessions. - Voice input (if used) is processed for speech-to-text; audio is not retained after transcription unless stated otherwise.

Technical data - Basic connection metadata (timestamps, session/WebSocket status) may be logged for debugging.

3. How data is used

  • To operate the chat and memory features (i.e. Aiko "remembering" past conversations).
  • To verify access tier (Patreon supporter vs. GitHub contributor).
  • To debug and improve the system.
  • Data is not sold, and is not shared with advertisers.

4. Where data is stored

  • Conversation logs and memory embeddings are stored locally on infrastructure operated by the developer (not a third-party cloud data warehouse), unless otherwise noted.
  • Because this runs on personal hardware rather than enterprise infrastructure, standard consumer-grade data-protection guarantees (redundancy, formal audits, SOC2, etc.) do not apply.

5. Third parties involved

  • Patreon and GitHub — used solely for OAuth login. Their own privacy policies govern the data they hold about you.
  • Any AI/ML APIs or model-hosting services used behind the scenes may process your messages to generate responses. [Add specific vendors here if you use hosted APIs alongside your local stack, e.g. image/video generation endpoints.]

6. Retention & deletion

  • Conversation history and memory data are retained to preserve the companion experience across sessions.
  • You may request deletion of your account data and memory history by contacting the developer at oppa.ai.org@gmail.com. Requests will be honored on a best-effort basis.

7. Your choices

  • You can stop using the service at any time.
  • You can request an export or deletion of your data.
  • Access can also be revoked by the operator at their discretion (see Terms of Service).

8. Security

  • Reasonable, good-faith efforts are made to secure data, but as a solo-maintained hobby project, no formal security certification or guarantee is provided.

This policy may change as the project evolves. Continued use after changes are posted constitutes acceptance of the updated policy.
